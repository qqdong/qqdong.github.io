项目管理全过程最佳实践（下）

发表于 2018-05-19 | 分类于 summary

上一篇 提到，参加了部门组织的项目管理培训，培训主题是「项目管理全过程最佳实践」，培训时间为1天，讲师是现代卓越的郑晓龙老师，整体感觉，教授氛围和互动方式都很轻松，让人印象深刻，有不少收获，分享给大家。

上一篇整理分享了教授方式、概述、定目标、拜真佛，这篇继续整理，本篇的分享点：

  • 编计划
  • 要资源

编计划

编写计划时，需不要团队成员参与，如果是你，你会采用下面哪种方式？
A：你写出项目的计划并发给成员，请他们了解并有效执行；
B：让团队成员分别计划，你再最终整合；

可以通过分析下面的情况作出最后的选择：

  • 管理者的成熟程度：如果有很多这方面的经验，有成功的解决方案，可以偏向A；
  • 执行者的成熟程度：如果任务最终执行者这方面的经验比较丰富，可以偏向B；
  • 计划的紧迫程度：如果很紧迫，没有太多时间让每个成员都参与进来，可以偏向A；
  • 成员的参与程度：如果想提高成员的参与感，可以偏向B；

确定计划或里程碑后，需要进行细化，对工作进行分解，按照目标、活动、任务的结构进行细分，可以通过常用的思维导图工具进行分析。

进行任务分解时，要考虑全面，注意下面几类工作：

  • 技术 + 管理类
  • 自己的 + 相关方的协调和对接
  • 单元类 + 整合类
  • 实现类 + 转移类，转移给其他人继续进行
  • 显性类 + 隐性类

任务分解完成后，需要对任务需要的时间进行评估，一般使用「甘特图」进行展示。

常用的估算方法如下：

  • 对比分析：对比之前类似的项目进行评估；
  • 细化分析：任务尽量细化，这样评估更准确；
  • 三点分析：分别进行最悲观估计（P）、最乐观估计（O）、最可能估计（M），最终结果=(P+4M+O)/6；
  • 专家分析：请专家、有经验的人进行核对；
  • 余量分析：根据确定的时间点，向前倒推；
  • 比例分析：不同阶段的比例一般是相似的，可以按比例进行评估；

时间评估之后，每个任务需要指定对应的负责人、执行者、知晓者和咨询者（专家），使用责任矩阵的方法进行整理。

A：负责人，R：执行者，I：知晓者，C：咨询者（专家）

最后，需要对项目成本和风险进行评估。

软件项目的成本主要是人力成本，可以根据时间评估进行估算。

列出风险时，要给出避免和减少风险的有效措施以及对应的责任人。

要资源

明确了目标和计划，如果人力或相关资源不够，需要向上级要，如果给了皆大欢喜，如果不给，可以站在领导的角度分析不给的原因，找到对应的措施：

  • 我也不足：保持好心态，用好现有资源；
  • 尝试性自信：表现能力，展现绩效；
  • 以为你够了：加强沟通，唤醒同情；
  • 激发你的潜力：用好现有资源，展现绩效；
  • 工作优先级的影响；尽力做，展现绩效；

执行过程中会出现各种问题，尽可能早的发现问题，发现问题的主要方式：

  • 汇报中，暴露问题；
  • 对标中，发现问题；
  • 走查中，感知问题；
  • 评审中，挖掘问题；
  • 风险中，预知问题；

会报告

一个阶段结束后，一般会向领导汇报，按照下面的思路，会让报告更完整、更流畅：

  • 面向目标谈结果;
  • 面向结果谈偏差；
  • 面向偏差谈原因；
  • 面向原因谈改进；
  • 面向改进谈支持；
  • 面向支持谈预测；
  • 面向预测谈承诺；
  • 面向承诺谈奖惩；

会总结

项目结束后，可以按照下面的思路进行反思：

  • 目标是如何达成的 ？
  • 项目成员和项目相关人员是如何参与项目的 ？
  • 该项目的成功之处是哪些 ？
  • 在下次项目中有哪些方面应当改进 ？
  • 收集个人感想、体会

一页纸项目管理

项目管理最重要的是沟通，90%以上的时间都花在沟通上，兼顾着横向和纵向的沟通，横向牵涉到各部门、外包公司等，纵向牵涉到对上沟通和对下沟通，
急需一页纸协助沟通和推进。

一页纸项目管理是一种工具，是向公司高层管理者或相关人反映项目状况和各方面负责人的绩效的一种载体，有效的沟通工具，一页纸是一份矩形图，连接
项目组成的各部分，各组成部分的所有者也被标注在文件上，项目进展也被不同颜色区分，能更好的把握情况。

老师给了模板和案例，以最近我负责的一个需求「智能客服系统多租户多时区改造」为例，说说一页纸都有什么。

首先，明确项目负责人、项目目标、项目完成时间。

约定项目里程碑和计划完成时间。

细化任务，明确开始和结束时间、任务的责任人，任务是对应到里程碑的，在纸上标注他们的相关性，执行过程中，不断更新任务的状态

根据任务需要的时间，可以分析出时间和成本。

针对项目的风险，需要给出避免和较少风险的方案，以及对应的负责人。
